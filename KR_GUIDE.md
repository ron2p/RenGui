## 📘 RenGui 사용자 가이드
RenGui는 코딩 없이 클릭만으로 비주얼 노벨을 만들 수 있는 차세대 엔진입니다.   
이 문서는 RenGui 에디터를 사용하여 나만의 게임을 만드는 방법을 설명합니다.

---

## 🗂️ 1. 준비하기 (폴더 구조)
게임을 만들기 전에, 사용할 이미지와 음악 파일들을 정해진 폴더에 넣어야 합니다.   
* `assets/images`: 배경 이미지(.png, .jpg)와 동영상 배경(.ivf)을 넣으세요.   
* `assets/sprites`: 캐릭터 스탠딩 이미지(.png)를 넣으세요 (배경이 투명해야 합니다).   
* `assets/sounds`: 배경음악(.mp3, .wav)과 효과음을 넣으세요.   
* `assets/fonts`: 게임에 사용할 폰트(font.ttf)를 넣으세요.   

> [!WARNING]
> 파일명은 가급적 영어로 작성해주세요.   
> (예: bg_school.png, char_aris.png)

---

## 🎬 2. 에디터 사용법
에디터를 실행(wails dev)하면 다음과 같은 화면이 나타납니다.

### 2.1. 탭 메뉴 (Menu)
사이드바 상단에 두 개의 탭이 있습니다.   
   
• 🎬 제작 (Action): 시나리오 카드를 추가하고 편집하는 곳입니다.   
• ⚙️ 설정 (Config): 게임 제목, 해상도, 디자인을 설정하는 곳입니다.   

### 2.2. 카드 추가하기
타임라인에 액션을 추가하려면 왼쪽 버튼을 클릭하세요.   
   
• 💬 대사 추가: 캐릭터가 말하는 일반적인 장면을 만듭니다.   
• 🎬 영상 연출: 대사 없이 동영상만 재생되는 컷신(Cutscene)을 만듭니다.   
• 🌿 분기점: 플레이어가 선택지(A/B)를 골라야 하는 순간을 만듭니다.   

### 2.3. 카드 수정하기
추가된 카드의 연필 아이콘을 누르면 상세 편집창이 뜹니다.   
   
| 항목 | 설명 |
| :--- | :--- |
| 화자 | 말하는 사람의 이름입니다. (예: 선생님, 아리스) |
| 대사 | 실제로 출력될 텍스트입니다. |
| 캐릭터 배치 | 왼쪽, 중앙, 오른쪽에 등장할 캐릭터 이미지를 선택합니다. |
| 배경 이미지 | 뒤에 깔릴 배경을 선택합니다. .ivf 파일을 선택하면 움직이는 배경이 됩니다. |
| 오디오 | BGM은 계속 반복 재생되며, SFX는 한 번만 재생됩니다. |
| 실행 조건 | 특정 변수 조건이 맞을 때만 이 대사를 보여줍니다. (예: love >= 100) |

## 🎥 3. 동영상(.ivf) 사용법
RenGui는 가벼운 실행을 위해 VP8/IVF 포맷의 영상을 사용합니다.   
mp4 파일을 바로 넣으면 작동하지 않습니다.   
FFmpeg를 사용하여 변환하는 법:   
```bash
ffmpeg -i my_video.mp4 -c:v libvpx -f ivf -an assets/images/my_video.ivf
```
> [!TIP]
> 변환된 .ivf 파일을 assets/images 폴더에 넣으면 에디터에서 자동으로 인식합니다.   
> 소리(-an)는 제거하고, BGM으로 따로 재생하는 것을 권장합니다

## 4. 게임 커스터마이징 (개발 중)
[설정] 탭에서 게임의 외관을 바꿀 수 있습니다.   
• 해상도: 기본은 1280 x 720입니다. FHD(1920 x 1080)로 바꿀 수 있습니다.   
• UI 디자인: 대화창의 색상, 투명도, 텍스트 색상을 마음대로 변경해보세요.   
• 변경 사항은 [프로젝트 저장] 버튼을 눌러야 story.json에 반영됩니다.   

## 🚀 5. 게임 실행 및 배포
### 테스트 플레이
에디터에서 저장을 완료한 후, 플레이어를 실행하여 확인합니다.
```bash
go run ./cmd/player
```
### 배포하기 (Build)
게임을 배포하려면 하나의 파일로 만들어야 합니다.

```bash
# 윈도우용 빌드 (.exe)
cd cmd/player
go build -o MyGame.exe -ldflags "-H=windowsgui"
```
> [!IMPORTANT]
> 게임을 전달할 경우, 생성된 MyGame.exe 파일과 assets 폴더, story.json 파일을 함께 보내야 합니다.
